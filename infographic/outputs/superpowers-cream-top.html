概述

Overview

定位

为 AI 编码代理打造的可组合技能集和工作流系统，覆盖设计、计划、分工、实现、测试、审查全流程

核心理念

TDD 驱动、两阶段代码审查（规范合规 + 代码质量）、子代理分工执行、小步提交频繁自测

触发方式

代理自动检测开发任务阶段并触发对应技能，也可通过自然语言显式请求特定技能

推荐流程

Recommended Workflow — 8 Steps
1

Brainstorming — 需求澄清与设计

用自然语言描述需求，代理通过提问澄清需求，分段展示设计文档并等待确认，避免返工

帮我想想这个功能的设计
2

审核设计 — 确认方案

阅读代理输出的分段设计，逐段确认或请求修改，确认后再继续下一步

3

Writing Plans — 任务拆分

设计分解为 2-5 分钟的小任务，每个任务包含目标描述 / 文件路径与变更 / 完整代码片段 / 验证步骤

把这个设计分成小任务
4

Using Git Worktrees — 隔离工作区

在新分支创建隔离工作树，运行安装和测试命令，确保干净的测试基线，保持主分支不受污染

5

Subagent-Driven Development — 执行计划

每个任务启动子代理 → 子代理按规范自测(spec compliance) → 代码质量审查 → 提交并运行测试

开始执行计划
6

Test-Driven Development — TDD

始终遵循 RED → GREEN → REFACTOR 循环：写失败测试 → 实现代码 → 检查通过 → 重构并清理临时代码

写失败测试再实现
7

Code Review — 质量审查

关键节点生成审查报告，按严重性分类（CRITICAL / HIGH 阻塞任务推进，MEDIUM / LOW 记录供后续处理）

请审查当前分支
8

Finishing Development Branch — 完成收尾

所有任务完成且测试通过后，提供选项：直接合并 / 发起 PR / 保留分支 / 放弃并清理工作树

完成分支并给出合并策略

技能库

Skills Library — 8 Core Skills

🤝 Brainstorming

编码前的设计与提问环节，分块展示设计方案并等待用户确认，确保理解一致后再开始实现

🌿 Using Git Worktrees

创建隔离工作树，在新分支上设置项目并验证测试基线，保持主分支干净不受未完成工作污染

📋 Writing Plans

将设计拆解为 2-5 分钟的小任务，输出每个任务的文件路径、完整代码变更和验证步骤

⚡ Subagent-Driven Development

为每个任务启动子代理执行，进行两阶段审核（先对照规范，再检查代码质量），小步提交频繁自测

🧪 Test-Driven Development

强制 RED-GREEN-REFACTOR 循环：写失败测试 → 实现通过 → 重构，并移除任何临时/多余代码

🔍 Requesting Code Review

在关键节点生成代码审查报告，识别严重问题并阻止进度直到修复，同时记录普通问题供后续处理

📥 Receiving Code Review

接收并处理来自其他代理或人类的审查反馈，按严重性排序并逐项解决，直到所有关键问题关闭

🎯 Finishing Development Branch

验证测试通过，提供最终选项：直接合并、发 PR、保留分支继续开发、或放弃并清理工作树
